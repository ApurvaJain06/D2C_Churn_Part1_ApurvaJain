# D2C_Churn_Part1_ApurvaJain
# D2C Churn Analytics Portfolio - Phase 1

## 📌 Project Overview
This repository contains the Data Audit, Exploratory Data Analysis (EDA), and strategic retention planning for our Direct-to-Consumer (D2C) brand. The goal of this phase is to identify the core drivers of customer attrition and validate hypotheses using data-driven insights.

---

## 📁 Repository Structure
* **`eda_audit.ipynb`**: Interactive Google Colab notebook containing data cleaning, relational joins, and the 6 required analytical charts.
* **`data_quality_report.md`**: Technical audit detailing dataset health, schema properties, and verification checks.
* **`business_memo.md`**: Executive facing brief outlining validated churn risk patterns and actionable retention strategies.
* **`requirements.txt`**: Python environment dependencies configuration.

---

## 📊 Core Insights Discovered
1. **Service Friction Impact:** Customers who churn display a significantly higher average number of customer support tickets, marking customer service resolution as a critical friction point.
2. **High-Value Vulnerability:** Attrition is actively impacting premium, high-spending cohorts rather than just discount seekers, emphasizing an urgent need for targeted retention.
3. **Systemic Attrition:** Churn rates remain proportionally uniform across all product categories (Skincare, Haircare, Makeup) and age brackets, indicating platform-level or operational gaps rather than product quality deficiencies.
